# Recycle-decision-chain defect hunt — findings

**Scope verified:** 8 chain files + 4 bats suites read fully; live state censused (`~/.claude/state/waiting-recycle`, `boundary-latch`, `idl.jsonl` 370k lines, `/tmp/cc-telemetry`). ## DEFECT ROWS

**[S1] Size-blind recycle chain — the OOM-dangerous session gets no recycle AND no page (total silence)**
- **File:** `hooks/waiting-recycle.sh:439,476` · `hooks/lib/context-econ.sh:83` · writer `statusline.sh:212`
- **Mechanism:** Every trigger keys on `used_pct` (context-window fill = `input_tokens/window`), which compaction/caching resets — *orthogonal* to the OOM driver, cumulative transcript size (65 MB/16,963 records per the diagnosis). Worse than "no recycle": the **entire** advisory/Stage-2/wedge-page/busy-page apparatus sits *downstream* of the `below-threshold-no-tell` abstain at L476, so a huge-transcript session at low `used_pct` exits at L476 and never reaches even the out-of-band operator page. The velocity layer can't save it either — `ce_burn` slopes `used_pct` (L83), and `ce_sample` **resets the history on any >2-pt fill drop** (L43-46), so a frequently-compacting giant shows perpetual `0 -1` "unknown forecast" → forecast-early never fires.
- **Failure scenario:** Session grows to 60 MB / low `used_pct` (heavy caching). Every PostToolUse poll → L476 abstain "below-threshold-no-tell". No recycle, no /compact, no wedge page, no IDL "escalated" — dead silent until the per-process OOM. This is the operator's actual crash.
- **Evidence:** `over_threshold=0; { [ "$fresh" = 1 ] && [ "$used" -ge "$eff_idle" ]; } && over_threshold=1` (L439); `{ [ "$over_threshold" = 1 ] || [ "$rot_valid" = 1 ]; } || abstain "below-threshold-no-tell..."` (L476, all page branches are L604-636, downstream). Live: `{"used_pct":15,"input_tokens":145656}`; idl `95× below-threshold-no-tell`, `0 fired`. grep for `stat -f%z|du|transcript.*size` across `hooks/ scripts/ bin/` → no size-triggered lifecycle action exists anywhere.
- **Fix sketch:** `statusline.sh` already reads `transcript_path` (L62,184) — add `transcript_kb=$(stat -f%z …)` to the telemetry JSON; OR a transcript-size threshold into the L476 trigger (`used ≥ eff_idle` **OR** `transcript_kb ≥ SIZE_MAX`), so the existing SAFE-gate/drain/page machinery engages on size.
- **Class:** leak→crash · **Status:** KNOWN(diagnosis doc §"recycle is blind to size") — the *"page-net is also downstream → zero signal"* and *compaction-resets-the-forecast* sub-mechanisms are NEW · **Confidence:** empirical

**[S2] A single stray untracked file pins the desk BUSY-soft forever → the proactive idle free-win never fires**
- **File:** `hooks/waiting-recycle.sh:494`
- **Mechanism:** SAFE classification holds soft on *any* `git status --porcelain` output — which includes untracked `??` files. A monitoring desk that ever leaves one untracked/scratch file in its cwd is permanently classified BUSY-soft, so it never takes the Tier-1 idle recycle (the whole "recycle at 35% while just-waiting = free win" point); it only ever pages at ≥75% (T_BUSY).
- **Failure scenario:** Desk drops a `/tmp`-style note or a log in its checkout → every poll → `dirty-tree-hold` (idl already shows this class live) → idle recycle silently downgraded to a busy escalation path.
- **Evidence:** `[ -z "$(git -C "$CWD" status --porcelain 2>/dev/null)" ] || hold soft "dirty-tree-hold"` (L494) — porcelain lists untracked by default.
- **Fix sketch:** Classify only *tracked* changes as the soft hold (`git status --porcelain --untracked-files=no`), or exclude gitignored/untracked noise from the idle-SAFE test.
- **Class:** stall · **Status:** NEW · **Confidence:** theoretical (checkout clean now; latent)

## Key-question answers (brief's explicit asks)

- **Size-blind trigger implemented anywhere?** No — verified by grep across `hooks/ scripts/ bin/`; only post-mortem `lead-crash-watchdog`/`cc-crash-report` record `transcript_kb`. The decision chain is 100% `used_pct`, and even the burn-forecast slopes `used_pct`. (S1)
- **Fire/suppressor pairs that deadlock into never-fire?** The dominant one is **not a suppressor pair but the trigger gate itself** (S1): below-threshold at L476 short-circuits everything including the page net. Secondary: a **permanently-dirty tree** (S2) blocks the idle path indefinitely, and a **stale cwd cooldown** (S2) can suppress a first recycle. The classic cap-trap and arm-clears-cooldown landmines are **defused** (see CHECKED-CLEAN).
- **State files lacking TTL/generation-keying?** All of `state/waiting-recycle/*` and `boundary-latch/*` — cwd-keyed leak is live (≈316 files), SID-keyed leak latent (S2-2/S2-3/S3-1).
- **Does the boundary drain run before the wall at high burn?** Yes for a *committed+green* Stop (forecast-early at L138-144) — but only if the session reaches Stop; a polling/looping desk never Stops (B-1, by design deferred to waiting-recycle, which is itself size-blind). **Stop hook firing "mid-tool-turn"**: not applicable — Stop fires at turn end; the mid-turn interrupt risk lives in waiting-recycle's PostToolUse Stage-2 exec (S2-4).
- **Can both fire simultaneously / what re-arms after a recycle?** Different events, no mutual latch (S3-2). After a LIVE recycle: fresh SID resets cap/escalate/fired/idlewatch; the cwd `cooldown-` is stamped at fire (L720) as the cross-generation loop-breaker; `arm-/brief-/live-` survive (successor must not re-arm, L707).
