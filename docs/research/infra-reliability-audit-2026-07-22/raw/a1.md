## DEFECT ROWS — session-OPENING surface, `scripts/handoff-fire.sh`

**[S1] FIRE-FAILED leaves a live, task-less pane orphaned — invisible to the reaper + duplicate-fire bait**
· File: `scripts/handoff-fire.sh:2114-2116`
· Mechanism: On engagement miss, the code prints "FIRE FAILED" and `exit 1` — but the pane it spawned is already live (a launched claude at an empty composer, or one that will engage seconds later). `ensure_registration`, `mark_fired_peer`, and `write_role` all sit on the SUCCESS branch (2105-2113), so the failed pane gets **no registry row, no fired-peer marker, no role**. It is invisible to `cc-reaper`/board and never GC'd.
· Failure scenario: Cold-worktree fire → slow install pushes first assistant turn past the 180s window → `exit 1`. The error tells the operator to "recover with a WARM re-fire", so they spawn a *second* session — meanwhile the first engages. Two live sessions on the task; the orphan is unreapable (no marker).
· Evidence: `echo "!! FIRE FAILED — never engaged … The pane is live but TASK-LESS … recover with a WARM re-fire" >&2` / `exit 1` (2115-2116); success-only guards at 2105/2109-2110/2113.
· Fix sketch: Before `exit 1`, either `it2 session close -f -s "$SPAWNED_PANE"` (kill the task-less pane) or write a provisional registry row + fired-peer marker so the reaper can GC it.
· Class: leak · Status: NEW · Confidence: empirical

· Mechanism: `"$POOL" claim` reserves a finite warm pre-provisioned worktree slot during CMD composition — which runs *before* `payload_lint_gate` (2094, can `exit 4`), `pre_trust` (2095), and `spawn` (2096, can `return 1`→`set -e` abort). None of those failure paths release the slot, and there is no `trap … EXIT`. `grep` confirms handoff-fire.sh only ever calls `claim`.
· Failure scenario: A wave of fires with malformed back-channel payloads each `exit 4` after claiming → the warm pool drains to zero → subsequent fires fall back to the slow cold path or stall. Slot is "claimed" forever.
· Evidence: `claimed="$("$POOL" claim "$WORKTREE" …)" … WT="$claimed"; WT_SETUP="pool"` (1541-1543); `grep -n 'release' scripts/handoff-fire.sh` → none; `grep -n 'trap '` → none.
· Fix sketch: `trap '[ "$WT_SETUP" = pool ] && [ -z "$SPAWNED_PANE" ] && "$POOL" release "$WORKTREE"' EXIT`, cleared on confirmed engagement.
