### DEFECT ROWS

**[S1] Identity-pin has a dead-pid hole: a recycled-in-place pane gets the successor collaterally closed, recorded as a clean teardown**
- **File:** `bin/cc-teardown:199-206` (pin), `:209-215` (idempotent short-circuit), `:257` (pane close)
- **Mechanism:** The pin refuses only when the resolved pid *differs* from the pinned pid, or when a *live* now-lstart differs. When the pinned pid is **dead**, `now_lstart` is empty so the lstart clause short-circuits (`:202` requires `[ -n "$now_lstart" ]`) and the pid clause matches (dead pinned pid == still-stale registry pid) → **no REFUSE**. The idempotent path (`:209-214`) only fires if the pane is *absent*; on a recycle-in-place the pane is *present* (successor occupies it), so control falls through to leg-2 `it2 session close` (`:257`), closing the successor's pane. Effect-verify then reads proc_gone=1 (old pid dead) + pane_gone=1 → records **TEARDOWN success** (`:270-272`).
- **Failure scenario:** cc-reaper classifies a `handed-off-lead`/`finished` with pid=1234 (alive) and pins `--expect-pid 1234`. Before its per-candidate act (sweeps run 246–302s, candidates processed serially), that session exits and a successor `/handoff --recycle`s **into the same pane** with new pid 5678, but the successor's SessionStart hasn't rewritten the registry yet. `resolve()` reads the stale row (pid 1234). Pin matches → no refuse → cc-teardown closes the pane the live successor is running in.
- **Evidence:** Pin comment itself states the flawed assumption — `:197` *"a dead expected pid (empty now-lstart) is NOT a recycle (that is the idempotent path below)"* — but the idempotent path requires `pp0=1` (absent), which a recycle-in-place violates. Test 13 (`cc-teardown:468-479`) only covers the registry-shows-*different-live-pid* case, never stale-same-dead-pid + present pane.
- **Fix sketch:** When a pin is supplied AND the pinned pid is dead AND the pane is still present, treat the occupant as indeterminate → REFUSE (fail-closed), or re-derive the pane's current live pid and pin against *that* before any close.
- **Class:** race/closure · **Status:** NEW · **Confidence:** theoretical (path proven; mitigated by cc-reconcile healing pre-sweep stale rows at `cc-reaper:345`, so only an intra-sweep recycle in the classify→act gap is exposed — but that gap is now minutes-wide).
